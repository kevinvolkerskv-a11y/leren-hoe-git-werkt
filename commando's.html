Commando's van git

Hieronder staan enkele veelgebruikte Git-commando's en hun beschrijvingen:

Git init, Dit commando wordt gebruikt om een nieuw Git-repository te initialiseren in een bestaande map. Wanneer je begint aan een nieuw project, gebruik je git init om een Git-repository op te zetten.

Git clone, Met dit commando kun je een bestaande Git-repository kopiÃ«ren naar je lokale machine. Het maakt een volledige kopie van de repository, inclusief alle bestanden, branches en geschiedenis.

Git add, Dit commando wordt gebruikt om wijzigingen in bestanden toe te voegen aan de staging area. Voordat je wijzigingen kunt committen, moet je ze eerst toevoegen met git add.

Git commit, Met dit commando kun je de wijzigingen die in de staging area zijn toegevoegd, vastleggen in de repository. Elke commit bevat een beschrijving van de wijzigingen en een unieke identificatie (hash).

Git push, Dit commando wordt gebruikt om lokale commits naar een externe repository te uploaden. Het synchroniseert je lokale wijzigingen met de centrale repository, zodat anderen ze kunnen zien en gebruiken.

Git pull, Dit commando wordt gebruikt om de laatste wijzigingen van een externe repository naar je lokale repository te downloaden. Het haalt de laatste commits en wijzigingen van de centrale repository en voegt ze toe aan je lokale repository.
